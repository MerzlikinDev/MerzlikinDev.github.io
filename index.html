Добро пожаловать в эмулятор НАМ

Машина Маркова, или Нормальный алгоритм Маркова (НАМ) - это одна из фундаментальных моделей в теории алгоритмов, которая формализует понятие «алгоритм» с помощью замены частей слов по заданным правилам Представьте, что у вас есть строка символов и набор простых правил, которые работают как команда «найти и заменить». Например, вы ищете в строке фрагмент «ab» и заменяете его на «c». Это и есть суть НАМ. Работа алгоритма строго регламентирована и выполняется пошагово: Задаётся алфавит - это набор допустимых символов, с которыми работает алгоритм. Задаётся список правил подстановки. Правила - это команды вида левая_часть → правая_часть. Они применяются к текущему слову строго по порядку. Выполнение начинается: На вход подаётся начальное слово. Алгоритм ищет первое подходящее правило и заменяет самое левое вхождение левой части на правую, создавая новое слово. Если правило было заключительным (обозначается как =>), то выполнение останавливается, и результат готов. Если правило было обычным, то процесс повторяется заново с начала списка правил, но уже для нового слова. Завершение работы: Алгоритм останавливается, когда: Будет применено заключительное правило. Не найдется ни одного правила, которое можно применить к текущему слову. Пример для наглядности Давайте рассмотрим простую задачу: избавиться от повторяющихся букв в слове, состоящем из букв a, b, c. Например, слово abbbcaa нужно превратить в abca. Для этого нам понадобятся следующие правила: aa → a bb → b cc → c Как это будет работать: Шаг 1: Просматриваем список правил. Первое правило aa → a не подходит, второе bb → b — подходит, так как в слове abbbcaa есть bb. Заменяем первое вхождение: abbbcaa → abbcaa. Шаг 2: Снова начинаем с первого правила. Правило aa → a по-прежнему не подходит. Правило bb → b подходит: abbcaa → abcaa. Шаг 3: Снова проверяем. Теперь подходит правило aa → a: abcaa → abca. Шаг 4: Просматриваем все правила. Ни одно из них не подходит к слову abca. Выполнение алгоритма завершено. в нашем эмуляторе вы загружаете txt файл с правилами в котором по строчкам они расписанны в формате левая_сторона ->/=> правая_сторона.
